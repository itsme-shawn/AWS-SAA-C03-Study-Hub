# Section 20: Data & Analytics

## к°ңмҡ”
AWSмқҳ лҚ°мқҙн„° л¶„м„қ м„ңл№„мҠӨ мғқнғңкі„лҘј лӢӨлЈЁл©°, мӢңн—ҳм—җм„ңлҠ” лҚ°мқҙн„° мҲҳм§‘, ліҖнҷҳ, л¶„м„қ, мӢңк°Ғнҷ” нҢҢмқҙн”„лқјмқёмқ„ м„Өкі„н•ҳлҠ” лҠҘл Ҙмқҙ мӨ‘мҡ”н•ҳлӢӨ. Athena, Redshift, OpenSearch, EMR, Glue, Lake Formation, QuickSight, MSK л“ұмқҳ м„ңл№„мҠӨ нҠ№м„ұкіј м Ғн•©н•ң мӮ¬мҡ© мӮ¬лЎҖлҘј кө¬л¶„н•  мҲҳ мһҲм–ҙм•ј н•ңлӢӨ.

## н•өмӢ¬ к°ңл…җ

### лҚ°мқҙн„° л¶„м„қ м„ңл№„мҠӨ н•ңлҲҲм—җ ліҙкё° (мҙҲліҙмһҗмҡ©)
> AWS лҚ°мқҙн„° л¶„м„қ мғқнғңкі„лҠ” мІҳмқҢм—” ліөмһЎн•ҙ ліҙмқҙм§Җл§Ң, м—ӯн• лЎң лӮҳлҲ„л©ҙ лӘ…нҷ•н•ҳлӢӨ:
> - **мҲҳм§‘:** Kinesis, MSK (Kafka) вҖ” мӢӨмӢңк°„ лҚ°мқҙн„°лҘј л°ӣм•„л“Өмһ„
> - **ліҖнҷҳ(ETL):** Glue вҖ” лҚ°мқҙн„°лҘј л¶„м„қм—җ л§һкІҢ к°Җкіө
> - **л¶„м„қ:** Athena (S3 м§Ғм ‘ SQL), Redshift (лҢҖк·ңлӘЁ лҚ°мқҙн„° мӣЁм–ҙн•ҳмҡ°мҠӨ), EMR (Hadoop/Spark л№…лҚ°мқҙн„°)
> - **кІҖмғү:** OpenSearch вҖ” н…ҚмҠӨнҠё л¶Җл¶„ кІҖмғү
> - **мӢңк°Ғнҷ”:** QuickSight (BI лҢҖмӢңліҙл“ң)
> - **кұ°лІ„л„ҢмҠӨ:** Lake Formation вҖ” лҚ°мқҙн„° л ҲмқҙнҒ¬ нҶөн•© кҙҖлҰ¬ л°Ҹ ліҙм•Ҳ

> **мҷң н•„мҡ”н•ңк°Җ?** S3м—җ мҢ“м•„л‘” лЎңк·ё нҢҢмқјмқҙлӮҳ CSV лҚ°мқҙн„°лҘј л¶„м„қн•ҳкі  мӢ¶мқҖлҚ°, DBм—җ л„Јкі  мӢ¶м§Җ м•Ҡмқ„ л•Ң. AthenaлҠ” S3 нҢҢмқјм—җ м§Ғм ‘ SQLмқ„ мӢӨн–үн•ңлӢӨ. м„ңлІ„ м—Ҷмқҙ мӮ¬мҡ©н•ҳкі , мҠӨмә”н•ң лҚ°мқҙн„° м–‘л§ҢнҒјл§Ң кіјкёҲлҗҳлҜҖлЎң к°„н—җм Ғ л¶„м„қм—җ л§Өмҡ° м Җл ҙн•ҳлӢӨ.

- **м„ңлІ„лҰ¬мҠӨ** SQL мҝјлҰ¬ м„ңл№„мҠӨ, S3м—җ м ҖмһҘлҗң лҚ°мқҙн„°лҘј лі„лҸ„ лЎңл”© м—Ҷмқҙ м§Ғм ‘ л¶„м„қ
- Presto(мҳӨн”ҲмҶҢмҠӨ л¶„мӮ° SQL мҝјлҰ¬ м—”м§„) кё°л°ҳ, н‘ңмӨҖ SQL мӮ¬мҡ©
- м§Җмӣҗ нҳ•мӢқ: CSV, JSON, ORC, Avro, Parquet
- **к°ҖкІ©**: мҠӨмә”лҗң лҚ°мқҙн„° TBлӢ№ $5.00 (м»¬лҹјнҳ• нҳ•мӢқ мӮ¬мҡ© мӢң мҠӨмә”лҹүмқ„ мӨ„м—¬ л№„мҡ© м Ҳк°җ)
- Amazon QuickSightмҷҖ м—°лҸҷн•ҳм—¬ лҰ¬нҸ¬нҢ…/лҢҖмӢңліҙл“ң
- мӮ¬мҡ© мӮ¬лЎҖ: BI/л¶„м„қ/лҰ¬нҸ¬нҢ…, VPC Flow Logs(л„ӨнҠёмӣҢнҒ¬ нқҗлҰ„ лЎңк·ё)/ELB Logs/CloudTrail(API к°җмӮ¬ лЎңк·ё) л¶„м„қ

#### Athena м„ұлҠҘ мөңм Ғнҷ”
- **м»¬лҹјнҳ• лҚ°мқҙн„°** мӮ¬мҡ© (Parquet, ORC) - мҠӨмә”лҹү к°җмҶҢлЎң л№„мҡ© м Ҳк°җ
- GlueлҘј мӮ¬мҡ©н•ҳм—¬ Parquet/ORCлЎң ліҖнҷҳ
- **лҚ°мқҙн„° м••м¶•** (bzip2, gzip, lz4, snappy л“ұ)
- **S3 нҢҢнӢ°м…”лӢқ**: к°ҖмғҒ м»¬лҹјмңјлЎң мҝјлҰ¬ мөңм Ғнҷ” (мҳҲ: year=1991/month=1/day=1/)
- **лҢҖмҡ©лҹү нҢҢмқј мӮ¬мҡ©** (128MB мқҙмғҒ) - мҳӨлІ„н—Өл“ң мөңмҶҢнҷ”

- **Data Source Connectors**: Lambdaм—җм„ң мӢӨн–ү (CloudWatch Logs, DynamoDB, RDS л“ұ)

### Amazon Redshift
> **AthenaмҷҖмқҳ м°Ёмқҙ:** AthenaлҠ” к°„н—җм ҒВ·мһ„мӢң мҝјлҰ¬м—җ м Ғн•©н•ң м„ңлІ„лҰ¬мҠӨ мҝјлҰ¬ м„ңл№„мҠӨ, RedshiftлҠ” ліөмһЎн•ң мҝјлҰ¬к°Җ л§Өмқј лҢҖлҹүмңјлЎң мӢӨн–үлҗҳлҠ” кё°м—… лҚ°мқҙн„° мӣЁм–ҙн•ҳмҡ°мҠӨм—җ м Ғн•©н•ҳлӢӨ. RedshiftлҠ” нҒҙлҹ¬мҠӨн„°лҘј н•ӯмғҒ к°ҖлҸҷн•ҳлҜҖлЎң кі м • л№„мҡ©мқҙ мһҲм§Җл§Ң, л№ҲлІҲн•ҳкі  ліөмһЎн•ң м§‘кі„/мЎ°мқё мҝјлҰ¬м—җм„ң AthenaліҙлӢӨ нӣЁм”¬ л№ лҘҙлӢӨ.
>
> - **OLAP(Online Analytical Processing):** лҢҖк·ңлӘЁ лҚ°мқҙн„°лҘј м§‘кі„В·л¶„м„қн•ҳлҠ” мӣҢнҒ¬лЎңл“ң. "м§ҖлӮң 3л…„к°„ м§Җм—ӯлі„ л§Өм¶ң н•©кі„" к°ҷмқҖ мҝјлҰ¬к°Җ н•ҙлӢ№. RDSмІҳлҹј мӢӨмӢңк°„ нҠёлһңмһӯм…ҳ(OLTP)мқҙ м•„лӢҳ.

- PostgreSQL кё°л°ҳмқҙм§Җл§Ң **OLTP(нҠёлһңмһӯм…ҳ мІҳлҰ¬)к°Җ м•„лӢҢ OLAP(лҢҖк·ңлӘЁ л¶„м„қ/лҚ°мқҙн„° мӣЁм–ҙн•ҳмҡ°м§•)**
- лӢӨлҘё лҚ°мқҙн„° мӣЁм–ҙн•ҳмҡ°мҠӨ лҢҖл№„ 10л°° м„ұлҠҘ, PB(нҺҳнғҖл°”мқҙнҠё) к·ңлӘЁ нҷ•мһҘ
- **м»¬лҹјнҳ• мҠӨнҶ лҰ¬м§Җ** (н•„мҡ”н•ң м»¬лҹјл§Ң мқҪм–ҙ л¶„м„қ мҝјлҰ¬ мөңм Ғнҷ”) + лі‘л ¬ мҝјлҰ¬ м—”м§„
- л‘җ к°Җм§Җ лӘЁл“ң: **Provisioned** (нҒҙлҹ¬мҠӨн„° нҒ¬кё° м§Ғм ‘ м§Җм •) лҳҗлҠ” **Serverless** нҒҙлҹ¬мҠӨн„°
- BI лҸ„кө¬ нҶөн•© (QuickSight, Tableau)
- **vs Athena**: мқёлҚұмҠӨ лҚ•л¶„м—җ лҚ” л№ лҘё мҝјлҰ¬/мЎ°мқё/м§‘кі„, н•ҳм§Җл§Ң нҒҙлҹ¬мҠӨн„° л№„мҡ©мқҙ л°ңмғқн•Ё

### Amazon EMR (Elastic MapReduce)
> **мҷң н•„мҡ”н•ңк°Җ?** Hadoop/Spark к°ҷмқҖ л№…лҚ°мқҙн„° н”„л Ҳмһ„мӣҢнҒ¬лҘј м§Ғм ‘ EC2м—җ м„Өм№ҳВ·мҡҙмҳҒн•ҳлҠ” кІғмқҖ ліөмһЎн•ҳлӢӨ. EMRмқҖ мҲҳл°ұ к°ң EC2 мқёмҠӨн„ҙмҠӨлЎң кө¬м„ұлҗң Hadoop/Spark нҒҙлҹ¬мҠӨн„°лҘј нҒҙлҰӯ лӘҮ лІҲмңјлЎң мғқм„ұВ·кҙҖлҰ¬н•ҙмӨҖлӢӨ. лҢҖк·ңлӘЁ лЁёмӢ  лҹ¬лӢқ лҚ°мқҙн„° м „мІҳлҰ¬, лЎңк·ё м§‘кі„, ліөмһЎн•ң ETL нҢҢмқҙн”„лқјмқё л“ұм—җ м Ғн•©н•ҳлӢӨ.

- **Hadoop(н•ҳл‘Ў) нҒҙлҹ¬мҠӨн„°** мғқм„ұ (мҲҳл°ұ~мҲҳмІң лҢҖмқҳ м„ңлІ„м—җ лҚ°мқҙн„°лҘј л¶„мӮ° м ҖмһҘВ·мІҳлҰ¬н•ҳлҠ” л№…лҚ°мқҙн„° н”„л Ҳмһ„мӣҢнҒ¬)
- Apache Spark(л¶„мӮ° лҚ°мқҙн„° мІҳлҰ¬), HBase(NoSQL), Presto(SQL), Flink(мҠӨнҠёлҰ¬л°Қ) лІҲл“Ө
- мҲҳл°ұ к°ң EC2 мқёмҠӨн„ҙмҠӨ нҒҙлҹ¬мҠӨн„° к°ҖлҠҘ
- мһҗлҸҷ мҠӨмјҖмқјл§Ғ + Spot Instances(м Җл ҙн•ң кІҪл§Өнҳ• мқёмҠӨн„ҙмҠӨ) нҶөн•©

#### EMR л…ёл“ң нғҖмһ… & кө¬л§Ө мҳөм…ҳ
- **Master Node**: нҒҙлҹ¬мҠӨн„° кҙҖлҰ¬/мЎ°м • - мһҘкё° мӢӨн–ү
- **Core Node**: нғңмҠӨнҒ¬ мӢӨн–ү + лҚ°мқҙн„° м ҖмһҘ - мһҘкё° мӢӨн–ү
- **Task Node** (м„ нғқ): нғңмҠӨнҒ¬л§Ң мӢӨн–ү - ліҙнҶө Spot
- кө¬л§Ө мҳөм…ҳ: On-Demand, Reserved (1л…„+), Spot
- **мһҘкё° мӢӨн–ү нҒҙлҹ¬мҠӨн„°** лҳҗлҠ” **мқјмӢңм Ғ(transient) нҒҙлҹ¬мҠӨн„°**

### AWS Glue
> **ETLмқҙлһҖ?** Extract(м¶”м¶ң) - Transform(ліҖнҷҳ) - Load(м Ғмһ¬)мқҳ м•Ҫмһҗ.
> - **м¶”м¶ң:** м—¬лҹ¬ кіі(S3, RDS, DB л“ұ)м—җм„ң лҚ°мқҙн„°лҘј к°Җм ёмҳҙ
> - **ліҖнҷҳ:** CSV вҶ’ Parquet, м»¬лҹјлӘ… ліҖкІҪ, н•„н„°л§Ғ, м •лҰ¬ л“ұ
> - **м Ғмһ¬:** ліҖнҷҳлҗң лҚ°мқҙн„°лҘј Redshift, S3 л“ұм—җ м ҖмһҘ
>
> GlueлҠ” мқҙ нҢҢмқҙн”„лқјмқёмқ„ мҪ”л“ң мөңмҶҢнҷ” + м„ңлІ„лҰ¬мҠӨлЎң мІҳлҰ¬н•ҙмӨҖлӢӨ.

- кҙҖлҰ¬нҳ• м„ңлІ„лҰ¬мҠӨ **ETL (Extract, Transform, Load)** м„ңл№„мҠӨ
- лҚ°мқҙн„° л¶„м„қмҡ© мӨҖл№„ л°Ҹ ліҖнҷҳ

### Q1. A company stores application logs in Amazon S3 in CSV format. They want to run ad-hoc SQL queries to analyze the logs without managing any infrastructure. The solution should be cost-effective. Which approach should they use?
**Options:**
- A) Load data into Amazon Redshift and query using SQL
- B) Use Amazon Athena to query data directly in S3
- C) Set up an Amazon EMR cluster with Hive
- D) Use Amazon RDS to import and query the data

**Answer:** B

**н•ҙм„Ө:**

> **л¬ём ң:** н•ң нҡҢмӮ¬к°Җ м• н”ҢлҰ¬мјҖмқҙм…ҳ лЎңк·ёлҘј Amazon S3м—җ CSV нҳ•мӢқмңјлЎң м ҖмһҘн•ҳкі  мһҲлӢӨ. мқён”„лқјлҘј кҙҖлҰ¬н•ҳм§Җ м•Ҡкі  мһ„мӢң SQL мҝјлҰ¬лЎң лЎңк·ёлҘј л¶„м„қн•ҳкі  мӢ¶лӢӨ. л№„мҡ© нҡЁмңЁм Ғмқё л°©лІ•мқҖ л¬ҙм—Үмқёк°Җ?

| м„ м§Җ | лІҲм—ӯ |
|------|------|
| A | Amazon Redshiftм—җ лҚ°мқҙн„°лҘј лЎңл“ңн•ҳкі  SQLлЎң мҝјлҰ¬ |
| B | Amazon AthenaлЎң S3мқҳ лҚ°мқҙн„°лҘј м§Ғм ‘ мҝјлҰ¬ |
| C | Apache HiveлҘј мӮ¬мҡ©н•ң Amazon EMR нҒҙлҹ¬мҠӨн„° м„Өм • |
| D | Amazon RDSм—җ лҚ°мқҙн„°лҘј мһ„нҸ¬нҠён•ҳкі  мҝјлҰ¬ |

**(A)** : RedshiftлҠ” лҚ°мқҙн„° мӣЁм–ҙн•ҳмҡ°мҠӨлЎң нҒҙлҹ¬мҠӨн„° н”„лЎңл№„м ҖлӢқкіј кҙҖлҰ¬к°Җ н•„мҡ”н•ҳлӢӨ. мқён”„лқј кҙҖлҰ¬ м—Ҷмқҙ мһ„мӢң мҝјлҰ¬лҘј мӢӨн–үн•ҳлҠ” мҡ”кө¬мӮ¬н•ӯм—җ л§һм§Җ м•ҠлҠ”лӢӨ. вҶ’ [рҹ“– Amazon Redshift](/section/20-data-analytics#amazon-redshift)

**(B) м •лӢө** : AthenaлҠ” м„ңлІ„лҰ¬мҠӨ SQL мҝјлҰ¬ м„ңл№„мҠӨлЎң S3м—җ м ҖмһҘлҗң лҚ°мқҙн„°лҘј м§Ғм ‘ л¶„м„қн•  мҲҳ мһҲлӢӨ. мқён”„лқј кҙҖлҰ¬к°Җ л¶Ҳн•„мҡ”н•ҳкі  мҠӨмә”н•ң лҚ°мқҙн„°м—җ лҢҖн•ҙм„ңл§Ң кіјкёҲлҗҳлҜҖлЎң л№„мҡ© нҡЁмңЁм ҒмқҙлӢӨ. вҶ’ [рҹ“– Amazon Athena](/section/20-data-analytics#amazon-athena)

**(C)** : EMRмқҖ Hadoop нҒҙлҹ¬мҠӨн„° м„Өм •кіј кҙҖлҰ¬к°Җ н•„мҡ”н•ҳлӢӨ. мһ„мӢң мҝјлҰ¬м—җлҠ” кіјлҸ„н•ң мҶ”лЈЁм…ҳмқҙлӢӨ. вҶ’ [рҹ“– Amazon EMR Elastic MapReduce](/section/20-data-analytics#amazon-emr-elastic-mapreduce)

**(D)** : RDSлҠ” OLTPмҡ© кҙҖкі„нҳ• лҚ°мқҙн„°лІ мқҙмҠӨмқҙлӢӨ. лЎңк·ё л¶„м„қ мҡ©лҸ„м—җ м Ғн•©н•ҳм§Җ м•ҠлӢӨ.

**н•өмӢ¬ к°ңл…җ:** Amazon Athena

**кҙҖл Ё л…ёнҠё:** [Amazon Athena](/section/20-data-analytics#amazon-athena)

### Q2. A data engineering team uses Amazon Athena to query large datasets in S3. They want to reduce query costs significantly. What is the MOST effective approach?
**Options:**
- A) Use larger EC2 instances for Athena
- B) Convert data to Apache Parquet format using AWS Glue
- C) Enable S3 Transfer Acceleration
- D) Use S3 Intelligent-Tiering storage class

**Answer:** B

**н•ҙм„Ө:**

> **л¬ём ң:** лҚ°мқҙн„° м—”м§ҖлӢҲм–ҙл§Ғ нҢҖмқҙ Amazon AthenaлЎң S3мқҳ лҢҖк·ңлӘЁ лҚ°мқҙн„°м…Ӣмқ„ мҝјлҰ¬н•ҳкі  мһҲлӢӨ. мҝјлҰ¬ л№„мҡ©мқ„ нҒ¬кІҢ мӨ„мқҙкі  мӢ¶лӢӨ. к°ҖмһҘ нҡЁкіјм Ғмқё л°©лІ•мқҖ л¬ҙм—Үмқёк°Җ?

| м„ м§Җ | лІҲм—ӯ |
|------|------|
| A | Athenaм—җ лҚ” нҒ° EC2 мқёмҠӨн„ҙмҠӨлҘј мӮ¬мҡ© |
| B | AWS GlueлҘј мӮ¬мҡ©н•ҳм—¬ лҚ°мқҙн„°лҘј Apache Parquet нҳ•мӢқмңјлЎң ліҖнҷҳ |
| C | S3 Transfer Acceleration нҷңм„ұнҷ” |
| D | S3 Intelligent-Tiering мҠӨнҶ лҰ¬м§Җ нҒҙлһҳмҠӨ мӮ¬мҡ© |

**(A)** : AthenaлҠ” м„ңлІ„лҰ¬мҠӨмқҙлҜҖлЎң EC2 мқёмҠӨн„ҙмҠӨлҘј м„ нғқн•ҳлҠ” к°ңл…җмқҙ м•„лӢҲлӢӨ. мЎҙмһ¬н•ҳм§Җ м•ҠлҠ” мҳөм…ҳмқҙлӢӨ.

**(B) м •лӢө** : AthenaлҠ” мҠӨмә”лҗң лҚ°мқҙн„° м–‘м—җ л”°лқј кіјкёҲ(TBлӢ№ $5)лҗңлӢӨ. Parquet/ORC м»¬лҹјнҳ• нҳ•мӢқмңјлЎң ліҖнҷҳн•ҳл©ҙ н•„мҡ”н•ң м»¬лҹјл§Ң мҠӨмә”н•ҳм—¬ л№„мҡ©мқ„ нҒ¬кІҢ м Ҳк°җн•  мҲҳ мһҲлӢӨ. Glue ETLмқ„ мӮ¬мҡ©н•ҳл©ҙ лҚ°мқҙн„°лҘј ParquetмңјлЎң мүҪкІҢ ліҖнҷҳн•  мҲҳ мһҲлӢӨ. вҶ’ [рҹ“– AWS Glue](/section/20-data-analytics#aws-glue)

**(C)** : S3 Transfer AccelerationмқҖ лҚ°мқҙн„° м „мҶЎ мҶҚлҸ„лҘј лҶ’мқҙлҠ” кІғмқҙлӢӨ. Athena мҝјлҰ¬ л№„мҡ©кіј м „нҳҖ л¬ҙкҙҖн•ҳлӢӨ.

**(D)** : S3 Intelligent-TieringмқҖ мҠӨнҶ лҰ¬м§Җ л№„мҡ© мөңм Ғнҷ”мқҙлӢӨ. Athena мҝјлҰ¬ л№„мҡ©кіј кҙҖл Ёмқҙ м—ҶлӢӨ.

**н•өмӢ¬ к°ңл…җ:** Athena Performance Improvement / AWS Glue

**кҙҖл Ё л…ёнҠё:** [Amazon Athena](/section/20-data-analytics#amazon-athena), [AWS Glue](/section/20-data-analytics#aws-glue)
